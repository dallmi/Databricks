# Multi-Channel Dashboard

Self-contained HTML dashboard that visualises communication activity across
Email (iMEP), Intranet News (AppInsights pageViews), and Events — joined
via the CammsTrackingID (see the project-level [`README.md`](../README.md)
for the join model).

## Files

| File | Purpose |
|---|---|
| [`multi_channel.html`](multi_channel.html) | Dashboard UI (Chart.js + vanilla JS, no build) |
| `seed.json` | Demo data generated by `scripts/generate_seed_data.py` |

## Run locally

Due to `fetch('seed.json')` the file cannot be opened as a `file://` URL.
Serve the `dashboard/` directory with any static server:

```bash
cd dashboard
python -m http.server 8000
# open http://localhost:8000/multi_channel.html
```

## Regenerate seed data

```bash
python scripts/generate_seed_data.py
```

## Wiring to live data

Replace `seed.json` with an export from the real facts. The dashboard
expects this shape:

```jsonc
{
  "generated_at": "2026-04-15T09:00:00Z",
  "packs": [
    {
      "pack_id":       "QRREP-0000058",
      "cluster":       "QRREP",
      "name":          "Q1 2026 Earnings Call",
      "publish_date":  "2026-03-05",
      "channels":      ["EMI", "INT", "EVT"],
      "audience":      48000,
      "theme":         "Financial Results",
      "topic":         "Quarterly Reporting",
      "target_region": "Global"
    }
  ],
  "email_events": [
    { "pack_id": "QRREP-0000058", "tracking_id": "QRREP-0000058-260305-0000100-EMI",
      "event": "opened", "count": 21500, "timestamp": "2026-03-05T10:00:00Z",
      "cluster": "QRREP", "channel": "EMI", "pack_name": "..." }
  ],
  "page_views": [
    { "timestamp": "2026-03-05T11:00:00Z", "page_id": "...", "page_name": "...",
      "site_id": "...", "site_name": "News", "time_on_page_sec": 98.3,
      "hr_division": "Division A", "hr_region": "EMEA",
      "tracking_pack_id": "QRREP-0000058", "tracking_cluster_id": "QRREP",
      "tracking_channel_abbr": "INT", "channel": "INT", "weight": 1 }
  ],
  "event_regs": [
    { "pack_id": "QRREP-0000058", "status": "attended", "count": 780,
      "event_datetime": "2026-03-19T14:00:00Z", "channel": "EVT",
      "tracking_id": "QRREP-0000058-260305-0000300-EVT" }
  ]

```

### Producing the same shape from Databricks

```sql
-- email_events: per (pack, event_type) aggregation
SELECT
  pack_id, tracking_id, event, sent_at AS timestamp,
  cluster, 'EMI' AS channel, pack_name,
  COUNT(*) AS count
FROM gold.fact_email
GROUP BY pack_id, tracking_id, event, sent_at, cluster, pack_name;

-- page_views: sample or aggregate fact_page_view
-- (use weight column if sampling to scale metrics back to population)

-- event_regs: per (pack, status)
SELECT pack_id, status, event_datetime, 'EVT' AS channel, tracking_id,
       COUNT(*) AS count
FROM gold.fact_event
GROUP BY pack_id, status, event_datetime, tracking_id;
```
