!2 A — Schema & TrackingId Location

!3 Q1 — Where does TrackingId live?
 * '''Date''': 2026-04-16
 * '''Scope''': tbl_email, tbl_email_links, tbl_analytics_link, tbl_email_receiver_status — search for columns tracking/camms/cpid/cplan.
 * '''Finding''': TrackingId exists '''only''' on imep_bronze.tbl_email (mailing level), not on engagement tables. Example: QRREP-0000058-240709-0000060-EMI. Cross-channel is a dimension, not a fact key.

!3 Q1b — Evaluate iMEP Gold layer
 * '''Date''': 2026-04-16
 * '''Scope''': Full schema + sample on imep_gold.tbl_pbi_platform_mailings and tbl_pbi_platform_events.
 * '''Finding''': Gold is mailing/event master with content & registration metrics — '''not''' a Send/Open/Click aggregate (Tier 2 carries those). Sentinel value 2124 for open-ended events.

!3 Q2 — Full core schemas
 * '''Date''': 2026-04-16
 * '''Scope''': DESCRIBE for TBL_EMAIL, TBL_EMAIL_RECEIVER_STATUS, TBL_ANALYTICS_LINK, TBL_EMAIL_LINKS with types + sample.
 * '''Finding''': 111 columns total, all nullable. tbl_email 54 cols, tbl_email_receiver_status carries LogStatus/TNumber/EmailLanguage, tbl_analytics_link 22 cols incl. Agent/LinkTypeEnum/EmailReceiverStatusId, tbl_email_links 10 cols (lean).

!3 Q3 — HR bridge TNumber ↔ GPN (first round)
 * '''Date''': 2026-04-16
 * '''Scope''': tbl_hr_employee, tbl_hr_user — identifier column inventory.
 * '''Finding''': '''Hypothesis falsified''' — no visible GPN column. T_NUMBER (lowercase) and UbsId (uppercase) are both T-numbers. Search was too narrow → opens Q3a/Q3b.

!3 Q3a — What IS the GPN in AppInsights?
 * '''Date''': 2026-04-17
 * '''Scope''': match 10 GPN sample values from AppInsights pageViews against HR columns (with string transformations).
 * '''Finding''': Superseded by Q3b — GPN is WORKER_ID in HR, no string transformation needed.

!3 Q3b — Extended HR search for the GPN column
 * '''Date''': 2026-04-17
 * '''Scope''': Three-step deep search across all HR tables — name tokens, 8-digit pattern counts, schema scan.
 * '''Finding''': imep_bronze.tbl_hr_employee.WORKER_ID '''is''' the GPN. The same table carries T_NUMBER too. Bridge = simple LEFT JOIN, no external source.

!2 C — TrackingId Format & Match Strategy

!3 Q23 — TrackingId format consistency iMEP ↔ SharePoint
 * '''Date''': 2026-04-20
 * '''Scope''': Structural comparison tbl_email.TrackingId ↔ sharepoint_bronze.pages.UBSGICTrackingID — length, case, segments, value overlap.
 * '''Finding''': Format identical (32-char 5-seg UPPER clean) '''but''' values nearly disjoint (Jaccard 0.004, 6/1677 overlap). SEG5 = channel (iMEP: EMI/NLI/TWE; SharePoint: IAN/ITS/OPN/ANC). '''Business-refined 2026-04-20: Cross-channel match is Pack-Level (SEG1-2) ONLY''' — SEG1-4 also fails because email and page of the same pack have different dates (SEG3) and activity sequences (SEG4).

!3 Q24 — TrackingId volume & adoption
 * '''Date''': 2026-04-20
 * '''Scope''': How many mailings actually have a TrackingId? Adoption over time + channel distribution.
 * '''Finding''': Only 986/73,930 mailings (1.3%) have TrackingId. Adoption ramps: 2024 = 99, 2025 = 637, 2026 YTD = 250. 80% are EMI. '''791 EMI rows → 696 distinct TIDs → 54 distinct Pack IDs (SEG1-2)'''. Dashboard universe = 54 packs, default time window from 2025.

!3 Q25 — UBSGICTrackingID distribution in pages
 * '''Date''': 2026-04-20
 * '''Scope''': Which SharePoint sites have tracking coverage, over which time spans, with which URL mapping?
 * '''Finding''': 99.4% of tracked pages on '''one single site''' ("News and events", 83 packs). Coverage rollout Sep 2024 (33.9%), peak 70.5% March 2026, never 80%. 1:1 URL↔TID mapping. Dashboard → restrict to News-and-events + floor 2024-09-01.

!2 D — Pipeline Lineage without Unity Catalog

!3 Q26 — Silver existence via schema naming
 * '''Date''': 2026-04-20
 * '''Scope''': SHOW SCHEMAS scan — which schemas end in _silver, which in _bronze / _gold?
 * '''Finding''': 17 _silver schemas exist. '''SharePoint uses the full Bronze→Silver→Gold''' (sharepoint_silver ~262M webpagevisited). '''imep_silver exists only for events''', email engagement skips Silver → imep_gold.final ~520M rows is the direct consumption endpoint. Medallion asymmetric. Stop planning silver.fact_email.

!3 Q27 — Join graph via column fingerprint
 * '''Date''': 2026-04-20
 * '''Scope''': Scan all table schemas × 8 schemas for EmailId/TrackingId/TNumber/GPN columns; reconstruct the join graph.
 * '''Finding''': 93 tables / 8 schemas mapped. Two full-key hubs: tbl_analytics_link (533M) + tbl_email_receiver_status (293M). EmailId in 12 tables, TNumber in only 2 (→ person analytics = email-grain only), TrackingId in exactly 4 and '''never co-occurring with EmailId''' → TrackingId is dimension, not fact. Cross-channel = TrackingId ↔ GICTrackingID via pages, '''not''' via engagement. New schema page_metadata_bronze discovered.

!3 Q28 — Pipeline hints from Delta History
 * '''Date''': 2026-04-20
 * '''Scope''': DESCRIBE HISTORY on the largest tables — write operations, Service Principal, operation types (MERGE/CTAS/Append).
 * '''Finding''': Unified MERGE → OPTIMIZE → VACUUM rhythm, 7-day retention. iMEP Bronze MERGE Upsert (tbl_analytics_link truly incremental 3-8K/run; tbl_email / tbl_email_receiver_status full-replace 27-72M/run). SharePoint Bronze: pages MERGE snapshot, pageviews Append WRITE (7 bursts = API pagination). iMEP Gold Full Rebuild — the 520M-row final is completely rewritten = biggest compute cost. All writes via Service Principal (SPN a71734ea-...). '''Scope limit''': concrete UTC refresh slots only visible in History, no complete scheduler view.

!3 Q29 — Gold tier structure from table shape
 * '''Date''': 2026-04-20
 * '''Scope''': Scan 52 Gold tables (iMEP + SharePoint) for row count, distinct-key count, column count; derive 4-tier hierarchy.
 * '''Finding''': Strict 4-tier hierarchy. '''Tier 0''' = atomic fact (imep_gold.final 520M grain mailing × recipient × event × hour; sharepoint_gold.pbi_db_interactions_metrics 84M). '''Tier 1''' = timespan × dim (15 tables). '''Tier 2''' = per-mailing summaries (incl. anomaly: engagement mixes emails+events with 117K vs 73K mailingIds). '''Tier 3''' = platform & reference dims (tbl_pbi_platform_mailings 73,920 rows, 927 TrackingIds). Cross-system bridge = dimensional via tbl_pbi_platform_mailings.TrackingId → pages → marketingPageId. SharePoint Gold has video sub-domain (3 tables, managed Declarative Pipeline).

!3 Q30 — Table properties & storage location
 * '''Date''': 2026-04-20
 * '''Scope''': DESCRIBE DETAIL / SHOW CREATE TABLE on 114 tables — ADLS paths, partitioning, formats, pipeline owners.
 * '''Finding''': 114 External Delta tables across 3 ADLS accounts (iMEP-bronze, SP-bronze, shared Gold). '''Gold is co-located''' → cross-channel joins possible inside Fabric/Spark.
