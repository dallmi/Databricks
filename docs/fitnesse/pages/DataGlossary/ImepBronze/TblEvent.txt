!1 TblEvent ('''imep_bronze.tbl_event''')

'''Event master table''' — events as a feature in iMEP (not to be confused with "email events" in the sense of Opens/Clicks). Carries '''TrackingId''' (one of only 4 tables with this column). For event registrations see additionally imep_silver.eventregistration (13.7M).

!2 Meta
| '''Layer'''          | Bronze |
| '''Source system'''  | iMEP (SQL Server) -> Change Data Capture (CDC) -> Delta Bronze |
| '''Grain'''          | 1 row per event definition |
| '''Primary key'''    | Id |
| '''Cross-channel key''' | TrackingId |
| '''Write pattern''' | MERGE |
| '''Rows'''           | ~100K (2009 - Apr 2026 — oldest dataset in the project!) |

!2 Key Columns (expected — exact verification via DESCRIBE)

| '''Column'''   | '''Type''' | '''Role'''           | '''Notes''' |
| Id             | string     | PK                   | Event GUID |
| TrackingId     | string     | '''Cross-channel key''' | Format like tbl_email.TrackingId, 32-char 5-seg |
| Title          | string     |                      | Event name |
| EventDate      | timestamp  |                      | When the event takes place |
| CreatedBy      | string     |                      | TNumber of the creator |
| CreationDate   | timestamp  |                      | When the event was created |

!2 Primary Joins

!3 To imep_silver.eventregistration — Who registered?

{{{
SELECT e.Title, e.EventDate, e.TrackingId, er.TNumber, er.RegistrationStatus
FROM   imep_bronze.tbl_event           e
JOIN   imep_silver.eventregistration   er ON er.EventId = e.Id
WHERE  e.TrackingId IS NOT NULL
  AND  e.EventDate >= '2025-01-01'
}}}

!3 To imep_gold.tbl_pbi_platform_events — Pre-aggregated with registration count

{{{
SELECT e.Title, e.TrackingId, pe.RegistrationCount
FROM   imep_bronze.tbl_event                 e
JOIN   imep_gold.tbl_pbi_platform_events     pe ON pe.Id = e.Id
}}}

For dashboard consumption: use Gold instead of Bronze -> Silver aggregation.

!2 Quality Caveats

 * '''EventDate can be 2124 (sentinel for "open-ended")''' confirmed. In filter logic guard with WHERE EventDate < '2100-01-01' if desired.
 * '''Oldest dataset in the project''' (from 2009). Pre-2020 data has little relevance for cross-channel analyses, but historical events exist.
 * '''TrackingId adoption''': most pre-2024 events have no TrackingId — cross-channel attribution only possible for newer events.
