!1 TblEmail ('''imep_bronze.tbl_email''')

The master table for all iMEP mailings. Single source of the '''TrackingId''' per mailing — every downstream engagement table (receiver_status, analytics_link, email_links) references back via EmailId = Id. Cross-channel analyses start here.

!2 Meta
| '''Layer'''         | Bronze                                                            |
| '''Source system''' | iMEP (SQL Server) -> CDC -> Delta                                 |
| '''Grain'''         | 1 row per Mailing (template definition, not per recipient)        |
| '''Primary key'''   | Id                                                                |
| '''Cross-channel''' | TrackingId (32-char, 5 segments)                                  |
| '''Refresh'''       | 2x/day @ 00:00 + 12:00 UTC (MERGE full-table upsert, Service Principal) |
| '''Rows'''          | ~145K (Nov 2020 - Apr 2026)                                       |
| '''PII'''           | CreatedBy = TNumber of author, indirectly identifying              |

!2 Primary Joins

!3 Join to tbl_email_receiver_status (1:N) — Sends / Bounces
{{{
SELECT e.TrackingId, rs.TNumber, rs.DateTime, rs.LogStatus
FROM   imep_bronze.tbl_email e
JOIN   imep_bronze.tbl_email_receiver_status rs ON rs.EmailId = e.Id
WHERE  e.TrackingId IS NOT NULL
}}}

!3 Join to tbl_analytics_link (1:N) — Opens / Clicks
{{{
SELECT e.TrackingId, al.TNumber, al.LinkTypeEnum, al.CreationDate AS event_time
FROM   imep_bronze.tbl_email e
JOIN   imep_bronze.tbl_analytics_link al ON al.EmailId = e.Id
WHERE  al.IsActive = 1
}}}

!3 Join to imep_gold.tbl_pbi_platform_mailings (1:1) — Pre-aggregated Master
Same Id, enriched with content metrics. Consumption-ready table.

!2 Quality Caveats

 * '''TrackingId NULL-rate''': Not every mailing is tracked. For cross-channel attribution set !- WHERE TrackingId IS NOT NULL -!. Only 986/73,930 mailings (1.3%) have TrackingId, but adoption ramping up fast (2024: 99, 2025: 637, 2026 YTD: 250).
 * '''TrackingId format''': Always 32 chars, 5 segments separated by '-', UPPER. SEG5 encodes system ownership, not medium — for cross-channel match use SEG1-4 only.
 * '''Cross-channel path''': TrackingId is a '''dimension''', not a fact key — never co-occurs with EmailId in the same table. Cross-channel goes via tbl_email.TrackingId <-> sharepoint_bronze.pages.UBSGICTrackingID, NOT via engagement rows.
 * '''CreationDate != send time''': For actual send time, use tbl_email_receiver_status.DateTime.
 * '''CreatedBy format''': lowercase 't######'. Joins against tbl_hr_user.UbsId need LOWER(UbsId).

!2 Lineage (Bronze -> Gold; No Silver for Email!)

Email engagement skips the silver layer entirely. iMEP has confirmed this as an architectural choice (Q26). imep_silver exists only for events (invitation, eventregistration, event), not email.

!2 Bronze -> Silver -> Gold Transformation Rule

Build pack-level dimension from tbl_email (for silver.dim_pack or ad-hoc queries):

{{{
SELECT
  SPLIT(TrackingId, '-')[0] AS cluster,
  SPLIT(TrackingId, '-')[1] AS pack_number,
  CONCAT(SPLIT(TrackingId, '-')[0], '-', SPLIT(TrackingId, '-')[1]) AS tracking_pack_id,
  MIN(CreationDate) AS pack_created_at,
  COUNT(DISTINCT Id) AS mailing_count
FROM imep_bronze.tbl_email
WHERE TrackingId IS NOT NULL
GROUP BY 1, 2, 3
}}}
