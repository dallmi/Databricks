!2 Five Core Findings Everyone Must Know

These five facts are the '''hard constraints''' against which all cross-channel analyses must be built.

!3 1. The cross-channel link is dimensional, not factual
tbl_email.TrackingId <-> sharepoint_bronze.pages.UBSGICTrackingID — '''never''' via engagement tables (pageviews, tbl_analytics_link) directly. TrackingId is a dimension, not a fact key.

!3 2. Only ~4% of pages have a UBSGICTrackingID
1,949 out of 48,419 pages. This means ~96% of SharePoint interactions are '''not''' attributable to a pack. The dashboard must make this explicit.

!3 3. TrackingId adoption has only been ramping up since 2024/25
Only 986/73,930 mailings (1.3%) have a TrackingId. Default dashboard time window: from 2025 onward.

!3 4. Write patterns: MERGE incremental (Bronze) vs. Full Rebuild (Gold)
 * '''iMEP Bronze''': MERGE Upsert. tbl_analytics_link is '''truly incremental''' (3-8K rows per run); tbl_email / tbl_email_receiver_status are full-table upserts (27-72M per run).
 * '''SharePoint Bronze''': pages via '''MERGE daily snapshot replace''', pageviews via '''Append-WRITE''' (7 bursts per run, API pagination).
 * '''iMEP Gold''': '''Full Rebuild''' — tables (including the 520M-row final) are completely destroyed and rewritten, not incrementally merged.

'''Caveat''': refresh cadence deliberately not documented — '''we don't have a complete job scheduler view''' (Delta History only shows write events, not the full schedule). Dashboards should be built resilient to whichever snapshot is current, not against concrete time windows. (scope-limited)

!3 5. Email skips Silver entirely
imep_silver exists only for events. For email engagement there is '''no silver layer''' — imep_gold.final is the direct consumption endpoint (denormalized, HR-enriched).

!3 6. Gold is classified into 4 tiers

Both gold schemas follow the same strict hierarchy:

 * '''Tier 0''' — Atomic Fact (largest tables): iMEP final 520M, SP pbi_db_interactions_metrics 84M
 * '''Tier 1''' — Pre-aggregated Timespans (Rolling Windows, Date aggregates)
 * '''Tier 2''' — Per-Mailing / Per-Page Summaries (UniqueOpens/Clicks, Engagement)
 * '''Tier 3''' — Platform & Reference Dimensions (tbl_pbi_platform_mailings, employeecontact, Calendar etc.)

'''Mental Model''': iMEP Gold = message-centric per-recipient; SharePoint Gold = page-centric analytics. TrackingId is the only conceptual bridge, and it lives at the dimension level (Tier 3).

!3 7. Storage architecture: Gold co-located, zero partitioning

All 114 Delta tables are External, across '''3 ADLS accounts''' (iMEP-Bronze, SP-Bronze, '''shared Gold'''). The shared-gold design enables cross-channel joins inside Fabric/Spark without cross-account auth. '''⚠️ No partitioning on any table''' — biggest structural performance gap. Queries on final or interactions_metrics without a time filter = full-scan risk.
